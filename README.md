# 🎲 Disco Skills Bot

Telegram-бот в стиле Disco Elysium: внутренние голоса-характеристики комментируют твои сообщения, бросают кубики и ведут живой диалог.

Отправь боту что угодно — текст, голосовое или видео-кружок — и несколько характеристик откликнутся, каждая со своим броском 2d6.

---

## Как это работает

Каждое сообщение обрабатывается тремя слоями:

1. **Выбор характеристик** — AI выбирает наиболее подходящие голоса из реестра.
2. **Бросок кубиков** — `2d6 + уровень_скилла vs сложность`.
3. **Генерация ответа** — каждый голос отвечает в своей манере, учитывая результат броска.

### Механика броска

```
Roll 2d6 + skill_level vs difficulty

⚀⚀ (дубль 1)  → КРИТИЧЕСКИЙ ПРОВАЛ (всегда)
⚅⚅ (дубль 6)  → КРИТИЧЕСКИЙ УСПЕХ (всегда)
итого >= сложность  → УСПЕХ
итого < сложность   → ПРОВАЛ
```

| Сложность | Порог |
|-----------|-------|
| Тривиальная | 6–7 |
| Лёгкая | 8–9 |
| Средняя | 10–11 |
| Сложная | 12 |
| Грозная | 13 |
| Легендарная | 14 |
| Героическая | 15 |
| Божественная | 16–17 |
| Невозможная | 18–20 |

## Команды бота

| Команда | Действие |
|---------|---------|
| `/start` | Приветствие и регистрация |
| `/scene <описание>` | Создать диалог-сцену по описанию |
| `/skills` | Показать текущие уровни характеристик |
| `/new` | Сбросить диалог и начать заново |

## Технологии

| Роль | Инструмент |
|------|-----------|
| Telegram | aiogram 3.x (webhook / polling) |
| HTTP сервер | FastAPI + uvicorn |
| База данных | PostgreSQL + Tortoise ORM |
| Миграции | aerich |
| Состояние диалога | Redis |
| LLM | OpenAI gpt-4o-mini |
| Транскрипт голоса | OpenAI Whisper |
| Зависимости | uv |

---

## Быстрый старт (локально)

```bash
# 1. Установить зависимости
make install

# 2. Создать .env
make env
# Заполни BOT_TOKEN, DATABASE_URL, REDIS_URL, OPENAI_API_KEY

# 3. Поднять postgres + redis и инициализировать БД (первый раз)
make db-init

# 4. Запустить бота в режиме polling
make dev
```

### Переменные окружения

| Переменная | Описание |
|-----------|---------|
| `BOT_TOKEN` | Токен Telegram-бота |
| `WEBHOOK_URL` | URL для webhook (пусто → polling) |
| `DATABASE_URL` | PostgreSQL DSN |
| `REDIS_URL` | Redis URL |
| `OPENAI_API_KEY` | Ключ OpenAI |
| `OPENAI_MODEL` | Модель (по умолчанию `gpt-4o-mini`) |
| `WHISPER_MODEL` | Модель транскрипции (по умолчанию `whisper-1`) |
| `ALLOWED_USER_IDS` | Telegram ID через запятую (пусто → все) |
| `MAX_SKILL_RESPONSES` | Сколько характеристик отвечает на одно сообщение (по умолчанию `4`) |

---

## Деплой (Railway)

1. Создай проект на [Railway](https://railway.app)
2. Добавь PostgreSQL и Redis плагины
3. Задай env vars из `.env.example`
4. Деплой происходит автоматически через `Dockerfile`

Бот запустится в webhook-режиме, если задан `WEBHOOK_URL`.

## Разработка

```bash
make install      # установить зависимости
make dev          # запустить локально (polling)
make lint         # ruff check + format
make db-migrate MSG="описание"  # создать миграцию
make db-upgrade   # применить миграции
make db-reset     # пересоздать БД с нуля
```

## Структура проекта

```
main.py              — FastAPI app + aiogram lifespan
app/
  core/
    config.py        — настройки (pydantic-settings)
    database.py      — Tortoise ORM + Redis
  models/            — User, UserSkillLevel, Dialog, DialogNode
  data/
    skills.py        — реестр всех характеристик
  services/
    ai_service.py    — OpenAI: выбор скиллов, генерация ответов, транскрипт
    skill_service.py — механика бросков 2d6
    dialog_service.py— оркестрация диалога
  bot/
    handlers/        — /start, /scene, /skills, сообщения, коллбэки
    keyboards/       — инлайн-клавиатуры
    router.py        — главный aiogram Router
    states.py        — FSM состояния
```
