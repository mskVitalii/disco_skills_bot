Я повторяю концепт - ты его не уловил

1. Бот должен создавать истории из того, что пишет человек
2. Истории связанные с характеристиками
3. Должна быть маленькая ветка диалогов, при нажатии на любой из вариантов и вводе своего
4. Какая-то ветка диалога может содержать проверку на кубики. Если проверка неудачная - это тоже вариант

В качестве сообщения сейчас "..." - хотя это должны быть диалоговые опции


Отдельно логика парсинга сообщений
1. Я пишу сообщение с перечислением
2. Бот ловит это сообщение
3. Бот изменяет моё сообщение с кнопками / удаляет сообщение и пишет от моего лица (бизнес-аккаунта) новое - как угодно
4. При нажатии на кнопки
Это нужно всё, чтобы превратить мои сообщения в эти диалоги с характеристиками

Главная идея, что они должны быть подходящими под диалог
